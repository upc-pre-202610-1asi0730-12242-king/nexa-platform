# Nexa Platform v0.4.0 - IAM Contract Alignment & Database Seeding Refinement

## Overview
This release focuses on aligning the IAM authentication API contract with the frontend B2B web application. It introduces email-based login support, refines the database seeding service to support idempotent execution, and includes the initial Entity Framework Core migration and database schema snapshot.

## Main Changes

### Identity and Access Management (IAM)
- **Email-Based Login Support:** Modified `SignInResource` and updated the `AuthenticationController` to allow authentication using either the `Email` or `Username` property, resolving contract mismatches with the frontend login form.
- **Sign-in Mapping:** Ensured existing username-based client commands are still processed correctly via the `SignInCommandFromResourceAssembler` fallback mapping.
- **Sign-in Validation:** Added a Bad Request validator to reject requests that omit both Username and Email properties.

### Database Persistence & Migrations
- **EF Core Migrations:** Created the initial database migration (`InitialCreate`) and the complete schema snapshot of the database contexts.
- **Database Configuration:** Updated default database settings to run locally using snake_case conventions.

### Seed Data
- **Official Demo Accounts:** Updated `users.seed.json` to configure the three official quick access profiles (Elena Litano, Valeria Sanchez, Roberto Garcia) with the password `NexaDemo2026!`.
- **Idempotency Fix:** Enhanced `SeedDataService` to check if seeded users already exist, updating their fields/roles in-place if they differ, avoiding key conflicts and ensuring clean updates.

### Documentation
- Updated the main `README.md` file with explicit local database configuration and running instructions.
