# Nexa Platform v0.3.0 - Modular Backend Foundation

## Overview

This release establishes the first stable backend foundation for Nexa Platform. It introduces a modular monolith architecture based on Domain-Driven Design principles, organized around Nexa's official bounded contexts and prepared for PostgreSQL persistence, REST API exposure, auditability, localization, and frontend integration.

## Main Changes

### Platform Architecture

- Migrated the backend foundation to .NET 8.
- Organized the solution into Nexa's official bounded contexts:
  - CatalogManagement
  - Sales
  - Warehouse
  - Logistics
  - Invoicing
  - Iam
  - Shared
- Standardized the backend folder structure around Application, Domain, Infrastructure, and Interfaces.
- Added modular dependency injection per bounded context.
- Reduced direct service registration inside `Program.cs`.

### Persistence and Database

- Configured Entity Framework Core with PostgreSQL using Npgsql.
- Added a shared `AppDbContext` for the modular monolith.
- Added `ModelBuilder` configuration extensions per bounded context.
- Added Humanizer-based naming strategy for plural snake_case table names and snake_case columns.
- Documented the local PostgreSQL initialization workflow for `nexa_platform_db`.

### Shared Kernel

- Added base entity and auditable entity abstractions.
- Added audit timestamp handling through the EF Core save changes interceptor.
- Added base repository and unit of work infrastructure.
- Added global exception handling middleware.
- Added ProblemDetails support for REST error responses.
- Added CORS policy for local frontend integration.
- Added localized problem details messages through shared resources.

### Catalog Management

- Added the `CatalogItem` aggregate.
- Added value objects for catalog identity, product identity, item name, brand, category, stock, price, and cold-chain requirement.
- Added REST resources, assemblers, application services, repositories, and controller foundation.
- Prepared `/api/v1/catalog-items`.

### Sales

- Added `Order` aggregate and `OrderItem` entity.
- Added value objects requested for the sales workflow:
  - OrderNumber
  - CustomerId
  - ProductId
  - CatalogItemId
  - ItemName
  - Quantity
  - Money
  - PaymentConfirmation
  - InventoryReservation
  - RejectionReason
- Added order creation, confirmation, rejection, and cancellation workflow foundations.
- Prepared `/api/v1/orders`.

### Warehouse

- Added `InventoryItem` aggregate.
- Added stock reservation and release behavior.
- Added temperature range and warehouse location modeling.
- Prepared `/api/v1/inventory-items`.

### Logistics

- Added `Shipment` aggregate foundation.
- Added shipment status and temperature tracking value objects.
- Prepared `/api/v1/shipments`.

### Invoicing

- Added `Invoice` aggregate foundation.
- Added invoice generation and payment status workflow.
- Prepared `/api/v1/invoices`.

### Identity and Access Management

- Added `User` aggregate.
- Added sign-up and sign-in command flow.
- Added password hashing service.
- Added token generation service foundation.
- Added authentication resources and controller.
- Prepared `/api/v1/authentication/sign-up` and `/api/v1/authentication/sign-in`.

### Documentation

- Added XML documentation support.
- Added Swagger/OpenAPI documentation support.
- Updated backend architecture and bounded context documentation.

## Known Follow-up

- Generate and apply the initial EF Core migration once the .NET SDK and EF tooling are available in the environment:

```bash
dotnet ef migrations add InitialCreate \
  --project King.Nexa.Platform \
  --startup-project King.Nexa.Platform \
  --output-dir Migrations

dotnet ef database update \
  --project King.Nexa.Platform \
  --startup-project King.Nexa.Platform
```
