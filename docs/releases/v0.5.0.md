# Nexa Platform v0.5.0 - Custom Error Handling & Bounded Context Alignment

## Overview
This release focuses on establishing custom domain error handling across our core bounded contexts and aligning application services for Catalog Management, Invoicing, and Warehouse management.

## Main Changes

### Domain Error Handling
- **Custom Error Types:** Introduced `SalesError.cs` and `SalesErrors.cs` inside the Sales boundary, and `WarehouseError.cs` and `WarehouseErrors.cs` inside the Warehouse boundary to encapsulate operation failures cleanly.
- **Service Integration:** Updated application services to leverage these domain errors, improving diagnostic logs and API response reliability.

### Catalog & Categories
- **Service Refinement:** Aligned Category and Brand query and command application services to prepare for dynamic product catalogs in the frontend.

### Persistence
- **Database Schema Snapshots:** Verified that the initial DbContext conventions align with the local schema and prepare for upcoming migrations.
