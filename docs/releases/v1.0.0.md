This release represents **Version 1.0.0** of the **Nexa Platform**, prepared in coordination with the engineering team (**Gino Torrejón, Joaquin Verde, César Marín, Gerard Rojas**) led by **Diego Sandoval Yucra**.

## 🌟 Key Improvements in v1.0.0

### 1. 🗄️ Relational Persistence & Migration (PostgreSQL)
- **PostgreSQL Database Engine:** Migrated the entire persistence layer from MySQL to PostgreSQL (Npgsql) with EF Core 9.0 and .NET 10.0.
- **Convention-Driven Naming:** Configured pluralized snake_case database tables, columns, and foreign keys convention in AppDbContext.
- **Idempotent Local Seeding:** Created `seed-local.sql` preloading database catalog items, brands, categories, and warehouses with validation count scripts.
- **Seeding Service Safety:** Updated `SeedDataService` to check if seed data exists before execution to prevent duplicate key conflicts.

### 2. 🛡️ Security Isolation & Settings Hardening
- **Production Secrets Protection:** Isolated production database connection strings and JWT secrets from source control (now loaded through environment variables on Render).
- **Ignore Rules:** Excluded local developer settings (`appsettings.Local.json`, `.env`) using root `.gitignore`.

### 3. 🛡️ Repository Standards & Documentation
- **Security Policies & Contribution Rules:** Added GitHub standards directly to the code repository under `.github/`:
  - `SECURITY.md`: Outlining vulnerability disclosure paths and scopes.
  - `CONTRIBUTING.md`: Laying down coding conventions, branch structures, and PR rules.
  - `CODE_OF_CONDUCT.md`: Standardizing community expectations.
- **Modernized README:** Formatted standard badges, dynamic architecture tables, and project directories.
